# career · UI 设计稿

> 本文件定义职业时间线页面的具体界面设计。视觉语言遵循 `DESIGN.md`，行为与数据边界遵循 `AGENTS.md`。

## 设计目标

页面只展示职业时间线，其余内容全部去掉。

它不再承担身份介绍、关键指标展示、联系方式入口、简历下载入口等职责。简历已经负责“简洁履历”，这个页面只负责把职业经历按时间展开，让读者看到每一年做了什么、产出了什么、有什么价值。

## 信息架构

页面只有 1 个主体：

1. 职业时间线

职业时间线由两层组成：

1. 公司阶段
2. 年份事件

结构示例：

```text
职业时间线
├── 杭州南讯软件科技有限公司  2021.02 - 至今
│   ├── 2024  做了什么 / 产出与价值
│   ├── 2022  做了什么 / 产出与价值
│   └── 2021  做了什么 / 产出与价值
├── 杭州十二章纹文化传播有限公司  2019.07 - 2021.02
│   ├── 2020  做了什么 / 产出与价值
│   └── 2019  做了什么 / 产出与价值
└── 浙江博澜文化股份有限公司  2016.04 - 2019.04
    ├── 2019  做了什么 / 产出与价值
    └── 2017  做了什么 / 产出与价值
```

## 页面内容

### 页面标题

仅保留一个极简标题：

- 职业时间线

标题不承担个人介绍功能，只告诉读者当前页面是什么。

### 公司阶段

每家公司展示：

- 公司全称
- 任职时间
- 阶段定位

阶段定位只保留一句话，例如：

> 电商 SaaS、会员运营与互动营销产品，从功能开发走向前端系统负责人。

### 年份事件

每个年份事件展示：

- 年份
- 主题
- 做了什么
- 产出 / 价值

字段解释：

- 年份：`YYYY`，必要时可以写 `YYYY.MM`
- 主题：当年最重要的一句话
- 做了什么：项目、职责、动作
- 产出 / 价值：沉淀、影响、复用、规模、业务价值

## 布局

### 桌面端

桌面宽度 `>= 960px` 时：

- 页面最大宽度：`1080px`
- 主体为单一时间线
- 公司阶段采用两列：
  - 左侧：公司任职时间
  - 右侧：公司信息与年份事件
- 年份事件内部采用两列：
  - 左侧：年份
  - 右侧：主题、做了什么、产出 / 价值

桌面端不出现顶部身份区、指标区、页脚区。

### 移动端

移动宽度 `< 768px` 时：

- 单列展示
- 公司时间移动到公司名下方
- 年份事件仍然保持“年份 -> 内容”的垂直结构
- 每条事件之间保留明显间距

## 视觉规则

- 页面背景：`#FAF9F5`
- 公司阶段边框：`#E4DFD4`
- 年份事件背景统一为 `#F2EFE7`
- 暂不展示重点事件强调线，所有年份事件保持完全一致的卡片样式
- 不使用阴影、渐变、插画、图标或装饰元素
- 年份使用等宽字体
- 公司名使用衬线体
- 正文使用无衬线体

## 内容扩充规则

新增内容只新增年份事件，不改变页面结构。

新增一条年份事件时，使用以下结构：

```html
<article class="year-event" id="nascent-2025">
  <div class="event-year">2025</div>
  <div class="event-body">
    <h3>一句话主题</h3>
    <section>
      <div class="field-label">做了什么</div>
      <ul>
        <li>动作 1</li>
        <li>动作 2</li>
      </ul>
    </section>
    <section>
      <div class="field-label">产出 / 价值</div>
      <ul>
        <li>结果 1</li>
        <li>结果 2</li>
      </ul>
    </section>
  </div>
</article>
```

扩充原则：

- 一年可以有多个事件；如果同一年事件太多，拆成 `2024.上半年`、`2024.下半年` 或具体月份。
- 不确定年份时先写 `⟨待核实⟩`，不要编。
- 只写职业事件，不写个人介绍、不写联系方式、不写简历入口。

## 验收清单

- 页面只展示职业时间线。
- 不展示身份区、指标区、联系方式、简历入口和页脚。
- 每家公司可以继续追加年份事件。
- 年份事件清楚分成“做了什么”和“产出 / 价值”。
- 360px 宽时不横向滚动。
- 锚点可以定位到公司或年份事件。
